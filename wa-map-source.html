What are Marker Groups and how to use them!

Marker groups allow you to bundle together multiple marker pins and image layers into a single group which you can overlay on your maps to create alternative views.

For example a map can have a group called "Geographic Map" which has pins for the most important geological formations (mountains, rivers, natural wonders etc) and also an alternative version of your map displaying the geology of that region, versus the original map which was displaying the political borders.

What are Layers and how to use them!

Layers are images that you can overlay on your map. These can be added into a marker group in addition to your marker pins or just as alternative views of your map's image.

Title

Minimum Allowed Zoom

How far out/away you can zoom out. This is a negative multiplier. The smaller the number, the further away you will be able to zoom. (-2 by Default)

Maximum Allowed Zoom

How far in/deep you can zoom in. 0 means zoom to the 1:1 dimensions of the map. Anything above 0 will mean that the image will start to get blurry. (0 by Default)

BBCode Tags Reference

The World Anvil Tags are a simple way of adding some spice to your content. It will allow you to add headers, images, links, make text bold, italic or underline it and many other cool things. In order to use it. Use the list of available BBCodes below in your text fields.

For example if you wanted to make part of your text bold you can use [b]This text is bold[/b]. Anything in between [b] and [/b] will become bold.

If you want to see them in action and more detail

Embed any Soundcloud audio into your page. The URL is the url of the track's page.

Use the above to show a card of a user's basic details.

Advanced

@...
Autocomplete link to any of your articles. Just use @ followed by a minimum of any 3 characters on the title of any of your articles.
e.g. @Longsw (will autocomplete Longsword if an article exists and create a link to it.)
@TITLE(TEMPLATE)
Link to uncreated article.
Replace TITLE with the title you want to have
Replace TEMPLATE with the template you want to use. Check below for a list of available templates.
examples:
@Captain John Redbeard(person)
@Wickham Valley(location)
[breadcrumb]
Create a horizontal breadcrumb from your article to the world's main page. We suggest you use it under the Sections > Header > Credits. It only works on articles. It will only detect 2 levels of sub articles.
[toc]
Inject your world's full Table of Contents.
[articletoc]
Add this to have an automatically created Table of Contents based on the headers of your article
[roll:1d20+3]
[roll:1d20+3|Roll Initiative]
Rollable dice! You click on the Roll it will appear on the bottom right of the screen. Click again to remove.
[tooltip:TOOLTIP TEXT]DISPLAYED TEXT[/tooltip]
Create a tooltip adding more information on the text inside of it. To be used for explanations and abbreviations.
[url:LINK]TITLE[/url]
Link to page
e.g. [url:http://www.google.com]Go to Google[/url]
or
[url:http:...]Go to a header on the page[/url]
You can also add |tab after the url to make it open in a new tab
e.g. [url:http://www.google.com|tab]Go to Google[/url]
or
[url:http:...]Go to a header on the page[/url]
You can also add |tab after the url to make it open in top frame
e.g. [url:http://www.google.com|parent]Go to Google[/url]
[quote]QUOTE|AUTHOR[/quote]
Quote and author
Author is optional, just don't add the "|"
To break your text in to multiple lines use the [br] BBcode
e.g. [quote]To be or not to be|Hamlet[/quote]
[img:IMAGEID]
Image from World's image gallery
Options:
[img:id] (Auto scale)
[img:id|left] (Float left)
[img:id|right] (Float right)
[img:id|center] (Align center)
[img:id|ANYABOVE|WIDTH] (example: [img:60449|right|200])
[img:id|ANYABOVE|WIDTH|nolink] (with no link to image page)
Replace IMAGEID with the ID of the image as shown on your image gallery
e.g. [img:20|left]
[imgblock:IMAGEID]
The image followed by it's title, art credits and excerpt in a block.
Options:
[imgblock:IMAGEID|#000000b0] background image cover (in case the image is transparent)
--KEY::VALUE--
Key/value lists
Use it to create lists of items with key and value like Age: 23, Str: 17 etc.
e.g. --STR::17--
[row]
[col] COLUMN-1-CONTENT [/col]
[col] COLUMN-2-CONTENT [/col]
[/row]
2-columns layout
By using the structure above you can create content which is separated in two parallel columns. It is very important that you don't forget the [row] and [/row] or this can cause chaos to your article's presentation.
[row]
[col3] COLUMN-1-CONTENT [/col3]
[col3] COLUMN-2-CONTENT [/col3]
[col3] COLUMN-3-CONTENT [/col3] [/row]
3-columns layout
By using the structure above you can create content which is separated in three parallel columns. It is very important that you don't forget the [row] and [/row] or this can cause chaos to your article's presentation.
[anchor|ID]
Use anchor in conjuction with [url=#ID]...[/url] To create a jumping point to that place of your text. Anchors are the targets for url tags with a #ID link
[h1|ID]
(Or any other header from 1 - 6). Use anchor in conjuction with [url=#ID]...[/url] To create a jumping point to that place of your text. Anchors are the targets for url tags with a #ID link
[spoiler] CONTENT | TITLE [/spoiler]
Anything in between it will be hidden and a "show spoiler" button will appear. When clicked it will display the hidden content.
[aloud] [/aloud]
A simple box to be used for "read-aloud" boxes mostly to be used for RPG adventure authoring.
[pin:IDENTIER]
Embed any pin icon that can be used in a map. You can see all pins and copy their codes by pressing Alt+Enter while editing a map.
[marquee] [/marquee]
Scrolling text marquee.
Parameters:
  • Behavior: scroll, slide or alternate
  • Direction: left, right, up, down
  • Pixels speed: 0 - *
Unordered Lists
Shows content as a bulletpoint list
- Item 1
- Item 2
-- Item 2.1 (one level in)
--- Item 2.1.1 (two levels in)
- Item 3
There is no limit to the length of a list
Ordered Lists
Shows content as a numbered list
[ol]
- Item 1
- Item 2
-- Item 2.1 (one level in)
--- Item 2.1.1 (two levels in)
- Item 3
[/ol]
The old system using [li]...[/li] is still usable but the above is much faster and recommended.
Articles List by Tag
Shows show a list of articles based a tag assigned to them
[tagged:tag_name|TYPE|HEADER]
TYPE: Set to either "list" or "block"
HEADER: Set a header to define the header of this list. If empty it will show the tag's name
e.g. [tagged:water|list|Water Articles]
Blocks List by Tag
Shows show a list of statblocks based a tag assigned to them
[taggedblocks:tag_name|HEADER]
HEADER: Set a header to define the header of this list. If empty it will show the tag's name
e.g. [tagged:equipment|Arms & Equipment]
[noparse] Any content including WABBC [/noparse]
Using noparse will allow you to print on the screen for the readers the actual WABBC you used. This is great if yu want to create a guide for your co-authors or the community.

Userstate Contols
Userstate controls allow you to manage who can see a piece of content based on their current state.
[userstate:loggedin] This will only show if the user on this page is logged in [/userstate]
[userstate:loggedout] This will only show if the user on this page is logged out [/userstate]
[userstate:follower] This will only show if the user on this page is logged in and follows your World [/userstate]
[userstate:notfollower] This will only show if the user on this page is logged in and DOES NOT follow your world [/userstate]
